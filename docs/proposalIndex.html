Project Summary:

For this project, we will create a Minecraft clone in C++ using the OpenGL Library. In the game, the player should be able to move around, and be able to manipulate blocks by destroying or placing them.


Problem Description:

We will tackle the problem of rendering a large number of entities (blocks) in a game world traversable by the player. Rendering the whole world at full detail all the time is inefficient, so we will implement optimization techniques to speed up our game. Some techniques we may employ are selective rendering and reducing the level of detail for far away entities. We will also render appropriate environment lighting and general environment aesthetics, by employing techniques like ray tracing and path tracing. Of course, we will also have basic “game” (more like sandbox) capabilities.

Goals and Deliverables:

Questions we hope to answer:

  1. How can we efficiently render a game world containing a large number of interactable entities?
  2. How can we illuminate the game world realistically?
  3. How can we make cubelike blocks look like real objects? (time permitting)
  4. How do we simulate a day/night cycle with respect to world lighting? (time permitting)

What we plan to deliver:

  • Playable game with an environment made of blocks
  • The player will be able to destroy and build blocks.
  • The player will be able to control a character that moves around the game world.
  • The game world will be an island that contains several types of blocks with different appearances.
  • The game world will render at acceptable speeds. (Question 1)
    • Metric: frames rendered per second, target >= 20
  • The game world will incorporate realistic lighting with an immobile light source. (Question 2)
    • Metric: visual judgment, feedback from test trials

What we hope to deliver:

  • Make the blocks look more realistic by implementing textures with bump mapping. (Question 3)
  • Include more types of blocks (e.g. rocks, dirt, lava)
  • Implement a day/night cycle with a sun that rises and sets (Question 4)
  • Be able to randomly generate game worlds according to parameters such as size, flatness, etc.
  • Make a game that’s enjoyable to play
    • Metric: feedback from test trials
  • Include background music
